# 智谱周报 AI 只读预览设计

## 目标

在不写入飞书 Wiki、不复制周报工作表、不发送群消息的前提下，使用智谱开放平台的 OpenAI 兼容接口，将指定周的日报统一事实数据生成周报单元格候选内容和可追溯依据。该预览用于验证模型效果、提示词、事实筛选和输出约束，验证通过后再接入周报实例表与群聊推送。

## 范围

本阶段提供一个显式执行的命令：

```bash
npm run weekly:ai-preview -- --start 2026-07-13 --end 2026-07-17
```

命令只读取当前配置组的日报统一事实表和周报模板定位信息，并在本地标准输出或指定输出文件中生成预览结果。它不得调用工作表复制或写入接口，不得上传海报，不得发送飞书消息，也不得写入周报实例表。

本阶段不替换现有周六海报定时任务。现有任务使用 `TemplateAiProvider` 的本地规则生成“重点事项、风险阻塞、待跟进事项”；其迁移、停用或改造另行处理。

## 数据口径

预览只使用满足以下条件的日报统一事实记录：

- 日报日期在命令参数 `--start` 和 `--end` 的闭区间内。
- `事实记录状态` 为“有效”。
- 成员姓名、直属上级和敏捷小组采用事实表中已经固化的当时归属，不重新用当前通讯录覆盖历史归属。
- 同一成员、同一日报日期、同一来源通道存在多条有效记录时，按来源时间取最新一条。
- 表单与群聊同时提交的冲突沿用事实层已经处理后的有效记录，不在 AI 层再次合并或自行判断。
- 内容为空或缺少事实依据时保持空白，不允许模型补全、推测或改变业务口径。

## 生成单元

数据先在本地按周报目标模块分组，再分别调用模型，避免将全部日报一次性发送给模型：

- 模块二按“敏捷小组”分组，生成“本周重点事项说明”和“下周工作计划”。每个目标对应一个单元格，内容条数不设上限，但必须来自输入日报。
- 模块三按周报模板中的部门管理板块分组，生成“本周工作进展”和“下周工作计划”。每类最多填三个目标单元格；不足三条时其余单元格为空。
- 核心指标不根据日报自动推断或填写；没有负责人提供的明确数据时保持空白。

本阶段复用周报模板定位器解析出的单元格白名单。模型只能返回白名单中的目标单元格，任何额外单元格都必须丢弃。

## 智谱接入

继续使用现有 `OpenAICompatibleProvider` 的接口形态，通过环境变量配置：

```dotenv
AI_PROVIDER=openai-compatible
AI_BASE_URL=https://open.bigmodel.cn/api/paas/v4
AI_API_KEY=<仅保存在本地或服务器环境中>
AI_MODEL=glm-4-flash-250414
```

API Key 不写入仓库、配置 JSON、日志或预览输出。请求使用低温度，并要求只返回 JSON。日志仅记录模型名、输入记录数、分组、耗时、返回状态和降级原因，不记录密钥。

## 输出格式

命令输出一个稳定的 JSON 对象，至少包含：

```json
{
  "mode": "read_only_preview",
  "weekStart": "2026-07-13",
  "weekEnd": "2026-07-17",
  "provider": "openai-compatible",
  "model": "glm-4-flash-250414",
  "groups": [],
  "cells": {},
  "evidence": {},
  "warnings": []
}
```

`cells` 保存通过校验的候选单元格值；`evidence` 按单元格列出使用的事实记录 ID、日报日期和成员姓名，便于人工核对；`warnings` 记录空分组、无匹配模板目标、模型降级或输出被过滤等情况。输出必须显式包含 `mode=read_only_preview`，防止与正式写入结果混淆。

## 校验与降级

模型返回后按以下顺序校验：

1. 响应必须能解析为 JSON 对象。
2. 只保留模板定位器允许的单元格。
3. 所有值转换为去除首尾空白的字符串。
4. 模块三每类最多保留三个非空值。
5. 无对应事实依据的内容置空并写入警告。

API Key 缺失、网络错误、超时、非成功 HTTP 状态、空响应或非法 JSON 时，命令不得写飞书或伪装成 AI 成功结果。它应返回非零退出码，并输出不包含敏感信息的错误摘要；已经取得的本地规则参考值可以单独标记为 `fallbackReference`，不得混入 `cells` 冒充模型结果。

## 测试策略

- 单元测试覆盖参数校验、日期闭区间、只取“有效”记录和同通道最新记录。
- 单元测试覆盖按敏捷小组及部门管理板块分组、空数据保持空白。
- Provider 测试覆盖智谱兼容请求、合法 JSON、非法 JSON、HTTP 失败和缺少 API Key。
- 输出校验测试覆盖单元格白名单、模块三最多三条、证据映射和敏感信息不落日志。
- 命令集成测试使用假的 Bitable、Sheet Locator 和 AI Provider，断言没有调用任何写入、复制、上传或消息发送接口。
- 最后运行完整 `npm test`，并在真实测试组织中手动执行一次只读命令核对输出，不开启定时推送。

## 后续群聊路由

任务 1 不发送群消息。后续联调三个小团队推送时，三个逻辑团队先统一映射到当前测试群，并在消息标题中标记模拟团队名称，避免需要创建三个测试群。

正式组织不要求项目管理员本人加入每个分管领导群。每个目标群只需由该群管理员将应用机器人加入群，并通过群内绑定命令或配置流程让机器人记录自身收到事件中的 `chat_id`。机器人未加入或无发送权限的群不得配置为正式推送目标，失败时进入运维告警而不是改发其他业务群。

## 验收标准

- 指定 2026-07-13 至 2026-07-17 可生成只读 JSON 预览。
- 输出内容可按目标单元格追溯到日报事实记录。
- 无事实依据不生成，模块三不超过三条，模型不能写白名单外单元格。
- 执行前后飞书周报工作表、周报实例表和群聊均无变化。
- 智谱失败时结果明确失败，不把本地规则结果标记为 AI 结果。
- API Key 不进入 Git、日志和输出文件。
